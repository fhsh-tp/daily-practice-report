## 1. 登入

學生帳號由老師或管理員事先建立。收到帳號（username，使用者名稱）與密碼後，即可登入系統。

**操作步驟：**

1. 開啟系統登入頁面。
2. 輸入使用者名稱與密碼。
3. 點選「登入」。
4. 登入成功後，系統會自動導向 Dashboard（總覽頁面）。

**注意事項：**

- 登入設有頻率限制（Rate Limit，速率限制）：每分鐘最多嘗試 10 次。超過限制需稍候再試。
- 若帳號或密碼錯誤，頁面會顯示錯誤提示。
- 登入狀態有效期為 24 小時，到期後須重新登入。

## 2. 加入班級

登入後，須先加入至少一個班級（Class），才能使用簽到、提交任務等功能。加入班級有兩種方式：

### 方式一：使用邀請碼（Invite Code）

老師會提供一組邀請碼。

1. 在 Dashboard 頁面找到「加入班級」功能。
2. 輸入邀請碼。
3. 系統驗證成功後，即加入該班級。

### 方式二：瀏覽公開班級

系統中標記為公開（Public）的班級，可直接瀏覽並加入。

1. 開啟公開班級列表。
2. 找到目標班級，檢視班級名稱與說明。
3. 點選「加入」。

**注意事項：**

- 每位學生可同時加入多個班級。
- 已加入的班級不可重複加入，系統會回傳提示。
- 已封存（Archived）的班級不會在學生的 Dashboard 中顯示。

## 3. Dashboard 總覽

Dashboard 是登入後的主要頁面，彙整所有班級的即時資訊。

### 頁面資訊一覽

| 區塊 | 說明 |
|------|------|
| **班級列表** | 顯示所有已加入且未封存的班級，含班級名稱、班主、成員人數 |
| **簽到狀態** | 每個班級顯示今日是否已簽到、簽到時間窗口是否開放 |
| **任務狀態** | 顯示今日是否有任務模板（Task Template，任務模板）、是否已提交、提交審核狀態 |
| **統計摘要** | 累計點數（Total Points）、徽章數量、練習報告提交總數 |
| **已獲徽章** | 展示目前擁有的所有徽章 |
| **近期活動** | 最近 20 筆活動紀錄，包含簽到、提交任務、獲得徽章等事件 |

### 任務提交狀態說明

- **pending**（待審核）：已提交，等待老師審核。
- **approved**（已通過）：老師已核准。
- **rejected**（已退回）：老師退回，需要補繳。

## 4. 每日簽到 (Daily Check-in)

簽到功能讓老師掌握學生的出席狀況。完成簽到即可獲得點數。

**操作步驟：**

1. 在 Dashboard 的班級卡片中，確認簽到視窗（Check-in Window，簽到時間窗口）為開放狀態。
2. 點選「簽到」按鈕。
3. 簽到成功後，頁面顯示簽到時間，狀態變為「已簽到」。

**查看簽到狀態：**

系統提供簽到狀態查詢，回傳以下資訊：

- `is_open`：簽到視窗是否開放中。
- `closes_at`：簽到視窗關閉時間。
- `already_checked_in`：今日是否已完成簽到。
- `reason`：若無法簽到，顯示原因說明。

**注意事項：**

- 每日每個班級限簽到一次。重複簽到系統會自動忽略。
- 老師可設定簽到開放的星期（Active Weekdays，有效簽到日）與時間窗口（例如 08:00 至 09:00）。若不在有效時段內，簽到按鈕不會出現或點選後會提示原因。
- 簽到成功會自動觸發點數發放與徽章評估（Badge Trigger，徽章觸發）。

## 5. 提交練習報告 (Submit Task)

每日練習報告是 DPRS 的核心功能。老師會為每個班級指派每日任務模板，學生依照模板填寫並提交。

**操作步驟：**

1. 在 Dashboard 點選班級的「提交任務」連結，或進入該班級的提交頁面。
2. 頁面顯示今日的任務模板，包含需要填寫的欄位（Fields）。
3. 依照各欄位要求填入內容。
4. 點選「提交」。
5. 提交成功後，頁面顯示成功訊息與獲得的點數。

**提交後的流程：**

- 提交後狀態為 `pending`（待審核），等待老師審核。
- 老師審核通過後狀態變為 `approved`，並可能發布至社群牆。
- 若老師退回（`rejected`），學生會收到退回原因與補繳期限（詳見第 6 節）。

**注意事項：**

- 若今日無任務模板，提交頁面會顯示「今日無任務模板」。
- 同一日同一任務模板僅能提交一次（被退回後可補繳）。
- 系統會執行欄位驗證（Submission Validator，提交驗證器）。驗證不通過會顯示具體錯誤訊息。
- 提交成功後自動觸發點數發放與徽章評估。

### 查看歷史提交

- **全部班級歷史**：在「學習歷程」頁面可檢視所有班級的提交紀錄，最近 50 筆。
- **單一班級歷史**：進入特定班級的歷程頁面，可檢視該班級的提交紀錄，最近 100 筆。

## 6. 補繳 (Resubmission)

當老師退回（Reject）提交後，學生可在補繳期限（Resubmit Deadline，補繳截止時間）內重新提交。

**操作步驟：**

1. 在 Dashboard 或學習歷程頁面，找到被退回的提交。
2. 點選進入退回詳情頁面（Rejection Detail），查看：
   - 退回原因（Rejection Reason）
   - 補繳截止時間（若老師有設定）
3. 回到該班級的提交頁面，系統會偵測到該筆退回的提交，允許重新填寫。
4. 填寫修正後的內容，點選「提交」。

**注意事項：**

- 補繳提交會記錄母提交（Parent Submission，原提交）的關聯。
- 老師退回時，系統會自動扣回（Revoke，撤銷）原本發放的提交點數。
- 若補繳提交再次通過審核，點數會重新發放。
- 補繳截止時間由老師在退回時指定，格式為 ISO 8601 日期時間。

## 7. 查看點數 (Points)

點數是 DPRS 遊戲化（Gamification）機制的核心貨幣。學生透過簽到與提交任務累積點數。

**操作步驟：**

1. 進入「我的點數」頁面。
2. 頁面顯示：
   - **目前餘額**（Balance）：所有點數交易的淨值。
   - **交易紀錄**（Transaction History，交易歷程）：每筆點數增減的明細。

**交易紀錄包含以下資訊：**

| 欄位 | 說明 |
|------|------|
| `amount` | 點數變動量（正數為獲得，負數為扣除） |
| `reason` | 變動原因（如 checkin、submission 等） |
| `source_event` | 觸發來源事件 |
| `created_at` | 交易時間 |

**點數來源：**

- **簽到**：每次成功簽到獲得簽到點數（預設 5 點，由老師設定）。
- **提交任務**：每次成功提交獲得提交點數（預設 10 點，由老師設定）。
- **退回扣回**：被退回的提交，其對應點數會被扣除。
- **老師手動調整**：老師可手動扣除（Deduct）或撤銷（Revoke）點數。

## 8. 查看徽章 (Badges)

徽章（Badge）是系統頒發的成就標記，可由系統自動觸發或老師手動頒發。

**操作步驟：**

1. 進入「我的徽章」頁面。
2. 頁面列出所有已獲得的徽章，每個徽章包含：
   - **圖示**（Icon）：徽章的視覺符號。
   - **名稱**（Name）：徽章名稱。
   - **說明**（Description）：徽章的取得條件或意義。
   - **頒發時間**（Awarded At）：獲得徽章的時間。
   - **頒發原因**（Reason）：老師手動頒發時的備註（若有）。

**徽章取得方式：**

- **自動觸發**：當學生完成特定條件（如連續簽到、提交累計達標等），系統會依據徽章觸發條件（Badge Trigger，徽章觸發器）自動頒發。
- **老師手動頒發**：老師可在管理介面中選擇學生並頒發特定徽章。

## 9. 排行榜 (Leaderboard)

排行榜依據累積點數排列班級內所有學生的名次。

**操作步驟：**

1. 進入班級的排行榜頁面。
2. 頁面顯示：
   - **名次**（Rank）：依點數由高至低排列。同分的學生共享相同名次。
   - **顯示名稱**（Display Name）：學生的顯示名稱。
   - **累積點數**（Points）：該學生在此班級的總點數。
   - **徽章數量**（Badge Count）：該學生獲得的徽章總數。

**注意事項：**

- 排行榜是否顯示由老師決定。若班級未啟用排行榜（`leaderboard_enabled` 為 false），學生會看到「此班級未啟用排行榜」的訊息。
- 系統另有跨班級排行榜（Cross-Class Leaderboard），彙總所有公開且啟用排行榜的班級中學生的總點數。

## 10. 獎品預覽 (Prizes)

老師可為班級設定獎品（Prize），學生可預覽可兌換的獎品與所需點數。

**操作步驟：**

1. 進入班級的獎品頁面。
2. 頁面列出所有可見的（Visible）獎品：
   - **名稱**（Title）
   - **說明**（Description）
   - **類型**（Prize Type，獎品類型）：`online`（線上獎品）或 `physical`（實體獎品）
   - **圖片**（Image）：獎品圖片（若有設定）
   - **所需點數**（Point Cost，兌換點數）

**注意事項：**

- 學生只能看到老師標記為「可見」的獎品。老師隱藏的獎品不會出現在列表中。
- 獎品僅供預覽，實際兌換流程由老師線下處理。

## 11. 社群牆 (Community Feed)

社群牆是班級內的動態牆，當老師核准提交後，系統會自動發布動態。學生也可以分享自己的提交至社群牆，並對其他同學的動態以 Emoji（表情符號）表達反應。

**瀏覽動態：**

1. 進入班級的社群牆頁面。
2. 動態以時間倒序排列，每則動態包含：
   - 作者顯示名稱
   - 內容預覽
   - 發佈時間
   - 其他成員的 Emoji 反應

**對動態表達反應（Reaction）：**

1. 在動態下方點選 Emoji 按鈕（預設為 :thumbsup:）。
2. 每位成員對同一則動態限一次反應。
3. 可移除已送出的反應。

**分享提交至社群牆：**

1. 選擇自己已提交的練習報告。
2. 點選「分享至社群牆」。
3. 動態發佈後，其他班級成員即可看到。

**刪除動態：**

- 僅可刪除自己發佈的動態。老師亦有權刪除任何動態。

**注意事項：**

- 社群牆僅限班級成員存取。非班級成員無法瀏覽或互動。

## 12. 個人設定

學生可自行修改顯示名稱與密碼。

**修改顯示名稱：**

1. 進入「個人設定」頁面。
2. 在顯示名稱欄位輸入新的名稱。
3. 點選「儲存」。
4. 顯示名稱不可為空白。

**修改密碼：**

1. 在「個人設定」頁面找到密碼區塊。
2. 輸入目前密碼（Current Password）。
3. 輸入新密碼（New Password）。
4. 點選「儲存」。
5. 密碼更新成功後，系統會清除目前的登入狀態，需使用新密碼重新登入。

**注意事項：**

- 修改密碼設有頻率限制：每分鐘最多 5 次。
- 新密碼需符合系統的密碼強度要求（Password Strength，密碼強度）。若不符合會顯示具體錯誤訊息。
- 使用者名稱（Username）、身份標籤（Identity Tag）、權限（Permissions）等欄位由管理員管理，學生無法自行修改。

## 附錄：常見問題

### 簽到按鈕沒有出現？

- 確認目前時間是否在老師設定的簽到時間窗口內。
- 確認今日是否為有效簽到日（老師可設定僅特定星期開放簽到）。

### 提交頁面顯示「今日無任務模板」？

- 表示老師尚未為今日指派任務模板。請聯繫老師確認。

### 提交後顯示衝突（Conflict）？

- 表示今日已有提交紀錄。同一任務模板每日僅能提交一次。

### 點數被扣除了？

- 可能原因：老師退回了已核准的提交、老師手動扣除點數、出席記錄被更正。可在點數交易紀錄中查看詳細原因。

### 排行榜看不到？

- 該班級的排行榜功能可能未啟用。請聯繫老師確認。
